┌─ REMINDER CREATION ENTRY POINTS ────────────────────────────────────────────┐
│                                                                              │
│  1. CronAgent                                                                │
│     File: OneValet/builtin_agents/cron/agent.py                            │
│     Tool: cron_add()                                                         │
│     Used for: "Remind me daily at 8am", "Schedule a task"                  │
│                                                                              │
│  2. TodoAgent  (set_reminder)                                               │
│     File: OneValet/builtin_agents/todo/tools.py                            │
│     Tool: set_reminder()                                                     │
│     Used for: "Remind me tomorrow", "Don't forget about..."                │
│                                                                              │
│  Both must set: delivery_mode="announce"                                     │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ STORAGE ───────────────────────────────────────────────────────────────────┐
│                                                                              │
│  OneValet PostgreSQL:                                                        │
│  ├─ cron_jobs          → Full job definition (JSON)                         │
│  ├─ cron_run_log       → Execution history                                  │
│  └─ Fields:                                                                  │
│     ├─ id, user_id, name                                                    │
│     ├─ schedule: { kind, at|expr|every_ms }                                │
│     ├─ state: { next_run_at_ms, last_run_status, last_delivered }          │
│     ├─ delivery: { mode, channel, webhook_url }                            │
│     └─ payload: { message | text }                                          │
│                                                                              │
│  koi-backend Database:                                                       │
│  └─ push_tokens       → { user_id, token, platform, created_at }           │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ SCHEDULING & FIRING ───────────────────────────────────────────────────────┐
│                                                                              │
│  CronService (OneValet/triggers/cron/service.py)                           │
│  ├─ Async timer loop runs continuously                                      │
│  ├─ Sleep interval: max 60 seconds (MIN_SLEEP_S=0.1, MAX_SLEEP_S=60)      │
│  ├─ Checks: if (now_ms >= job.next_run_at_ms) → fire                       │
│  ├─ Prevents concurrency: skips if job already running                      │
│  ├─ Exponential backoff on error: [30s, 60s, 300s, 900s, 3600s]           │
│  └─ Auto-cleans stuck jobs (timeout > 2 hours)                             │
│                                                                              │
│  CronExecutor (OneValet/triggers/cron/executor.py)                         │
│  ├─ Mark job as running: job.state.running_at_ms = now                     │
│  ├─ Run orchestrator with AgentTurnPayload(message=reminder_msg)           │
│  ├─ Capture: summary, error_msg, duration_ms                               │
│  ├─ Call: CronDeliveryHandler.deliver()                                    │
│  ├─ For "at" jobs: delete after successful run                             │
│  ├─ Recompute next_run_at_ms for recurring jobs                            │
│  ├─ Save to PostgreSQL (state + run_log)                                   │
│  └─ Timeout: default 120 seconds per job                                   │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ PUSH NOTIFICATION DELIVERY ───────────────────────────────────────────────┐
│                                                                              │
│  Step 1: OneValet → koi-backend (if configured)                             │
│  ────────────────────────────────────────────────────────────────           │
│  CronDeliveryHandler (OneValet/triggers/cron/delivery.py)                  │
│  ├─ Check: job.delivery.mode == "ANNOUNCE"                                │
│  ├─ Build: metadata = { cron_job_id, task_name, delivery_mode }           │
│  ├─ Route: to PushNotification channel                                      │
│  │          (in self._trigger_engine._notifications list)                   │
│  └─ Call: push_sender_callback(user_id, title, message, metadata)         │
│                                                                              │
│  ⚠️  ISSUE: PushNotification is NEVER created by default!                  │
│           Only CallbackNotification created if callbacks.notify_url set    │
│                                                                              │
│  Step 2: koi-backend → Expo                                                 │
│  ────────────────────────────────────────────────────────────────           │
│  NotificationService (koi-backend/services/notification.py)                │
│  ├─ Get user's notification channel: "push" | "sms" | "both"              │
│  ├─ If "push":                                                              │
│  │   ├─ Query: tokens = db_client.get_user_push_tokens(user_id)           │
│  │   ├─ Build messages: [{ to: token, title, body, sound: "default" }]    │
│  │   └─ POST to: https://exp.host/--/api/v2/push/send                     │
│  └─ If "sms" (Pro tier only):                                              │
│      ├─ Get phone from db                                                   │
│      └─ Send via SMS provider                                               │
│                                                                              │
│  Step 3: Expo → Device                                                      │
│  ────────────────────────────────────────────────────────────────           │
│  ├─ Expo routes based on platform (iOS → APNs, Android → FCM)             │
│  ├─ Message reaches device                                                  │
│  ├─ If app closed: shown in OS notification center                         │
│  └─ If app open: handled by koi-app notification listeners                │
│                                                                              │
│  Step 4: koi-app receives notification                                      │
│  ────────────────────────────────────────────────────────────────           │
│  services/push.ts + hooks/useNotifications.ts                              │
│  ├─ Foreground: notification arrived                                        │
│  │   └─ Inject into chat UI (addMessage to chatStore)                      │
│  ├─ User tap: notification tapped                                           │
│  │   └─ Navigate to chat screen (router.replace)                           │
│  └─ Background: shown in notification center (OS handles)                  │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ PUSH TOKEN LIFECYCLE ──────────────────────────────────────────────────────┐
│                                                                              │
│  1. koi-app startup (app/(tabs)/_layout.tsx or app.tsx)                    │
│     └─ Call: setupPushNotifications()                                       │
│        ├─ Request OS permissions                                            │
│        ├─ Get Expo token: N.getExpoPushTokenAsync()                        │
│        ├─ POST /api/push/register { token, platform }                      │
│        ├─ Configure handler: setNotificationHandler(...)                   │
│        └─ Store locally: _currentPushToken                                 │
│                                                                              │
│  2. koi-backend receives registration                                       │
│     └─ routes/push.py: POST /api/push/register                             │
│        ├─ Validate: platform ∈ ["ios", "android"]                         │
│        ├─ Save: db_client.save_push_token(user_id, token, platform)       │
│        └─ Log: "Push token registered for user X (platform)"              │
│                                                                              │
│  3. Token stored in database                                                │
│     └─ Available for: db_client.get_user_push_tokens(user_id)             │
│                                                                              │
│  4. Unregister on logout                                                    │
│     └─ teardownPushNotifications()                                          │
│        ├─ DELETE /api/push/unregister                                       │
│        └─ db_client.delete_push_token(user_id, token)                      │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ KEY MODELS & TYPES ────────────────────────────────────────────────────────┐
│                                                                              │
│  CronJob                          Schedule                                   │
│  ├─ id: string                    ├─ AtSchedule (one-shot)                 │

│  ├─ payload: Payload              └─ CronScheduleSpec (cron expr)          │

│  │  └─ SystemEventPayload                                                  │
│  │      └─ text: string           DeliveryConfig                           │
│  ├─ delivery: DeliveryConfig       ├─ mode: "announce" | "webhook" | "none"
│  ├─ state: CronJobState           ├─ channel: string (optional)            │
│  │  ├─ next_run_at_ms             ├─ webhook_url: string (optional)       │
│  │  ├─ last_run_at_ms             └─ conditional: bool                    │
│  │  ├─ last_run_status            DeliveryMode enum:                       │
│  │  ├─ running_at_ms              ├─ ANNOUNCE → notification channels      │
│  │  ├─ last_delivery_status       ├─ WEBHOOK → HTTP POST                  │
│  │  └─ last_delivered             └─ NONE → silent                        │
│  └─ delete_after_run: bool                                                  │

┌─ 🔴 CRITICAL GAPS ──────────────────────────────────────────────────────────┐
│                                                                              │
│  1. PushNotification NOT initialized in app.py                             │
│     → Only if callbacks.notify_url configured (CallbackNotification)       │
│     → Result: Push delivery never happens unless URL is set                │
│                                                                              │
│  2. No koi-backend endpoint for receiving push from OneValet               │
│     → OneValet posts to callbacks.notify_url but endpoint doesn't exist    │
│     → koi-backend /api/push/register only handles token registration       │
│                                                                              │
│  3. No retry logic for failed Expo pushes                                   │
│     → Failures logged but not retried                                       │
│     → If Expo is down, reminders silently lost                              │
│                                                                              │
│  4. Push token registration not validated in koi-app                       │
│     → Fire and forget - no error handling                                   │
│     → App may think it's registered but isn't                               │
│                                                                              │
│  5. No delivery tracking in koi-backend                                     │
│     → OneValet tracks last_delivery_status in CronJob                      │
│     → koi-backend has no push_delivery_log table                            │
│     → Difficult to debug: "Did the push send?"                              │
│                                                                              │

┌─ SEQUENCE: HOW TO CREATE & TEST A REMINDER ─────────────────────────────────┐
│                                                                              │
│  1. Setup (one-time):                                                        │
│     a. koi-app: Open app → setupPushNotifications() called                  │
│     b. Device: Grant notification permission                                │
│     c. Backend: Token saved in database                                     │
│                                                                              │
│  2. Create reminder:                                                         │
│     a. User: "Remind me in 5 minutes"                                       │
│     b. CronAgent: calls cron_add() with delivery_mode="announce"           │
│     c. OneValet: stores in PostgreSQL, computes next_run_at_ms             │
│                                                                              │
│  3. Wait 5 minutes:                                                          │
│     a. CronService timer loop fires                                         │
│     b. CronExecutor.execute() runs                                          │
│     c. CronDeliveryHandler.deliver() called                                │
│     d. PushNotification.send() called (if initialized!)                    │
│     e. NotificationService.notify() queries push tokens                    │
│     f. Expo API receives POST with token list                              │
│     g. Device receives notification                                         │
│     h. koi-app listener injects into chat or shows in center               │
│                                                                              │
│  4. Debug if no notification:                                                │
│     a. Check: OneValet config has callbacks.notify_url                     │
│     b. Check: User has push token in database                              │
│     c. Check: job exists: SELECT * FROM cron_jobs WHERE user_id = ?       │
│     d. Check: job fired: SELECT * FROM cron_run_log WHERE job_id = ?       │
│     e. Check: delivery_status in OneValet logs                             │
│     f. Check: Expo API response                                             │
│     g. Check: App has useNotifications() hook registered                   │
│                                                                              │
